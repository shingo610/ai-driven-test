# 模範解答パターン（新人伴走ダッシュボード図解）

## 成功する図解の構造（「聞けない」前提）

```
1. ヘッダー（グラデーション）
   └─ タイトル + 「新人伴走ダッシュボード｜決定→精算」

2. 導入（ミナ × 指導社員AI）
   └─ ミナ: 何から手を付ければいいか分からない不安
   └─ AI: 工程に当てはめて、次の一手を3つまでに絞る宣言

3. 次の一手（今日やること：最大3つ）
   └─ 期限・相手・完了条件を必ずセット

4. いまの位置（進捗トラッカー：決定→精算）
   └─ 7ブロック（決定/見積/手配/書類/請求/実施/精算）
   └─ 完了/進行中/要確認を色とバッジで

5. いまの状態（できている/未完の一覧）
   └─ 新人の入力（Excel貼り付け）を、工程に沿って整理
   └─ 「未完」は“次にやること”へリンク（または矢印）

6. 詰まりポイントの対話（ミナ × AI）
   └─ 「精算ってどこまで？」など、知識ゼロの疑問を代弁
   └─ AIが“完了条件”の言い方を提示

7. 確認すべき質問（顧客/仕入先/社内）
   └─ 送れる文面テンプレを必ず入れる
   └─ 「いま聞けない」場合の安全な仮置き案も添える

8. 用語解説ボックス（初出の略語・業務語）
   └─ 精算/原価/未収/バウチャー等

9. まとめ（ミナ × AI）
   └─ AI: 次の一手3つの再掲 + “揃ったら次へ”の判断基準
   └─ ミナ: 最初の一手の宣言（例: まず在庫確認から）

10. 目次（フローティング・デスクトップのみ）
```

---

## 「次の一手」作成のルール（迷子防止）

- 最大 **3つ** まで（4つ以上あるなら優先順位で削る）
- それぞれに必ず付ける
  - **期限**（分からなければ「仮」を置く）
  - **相手**（顧客/仕入先/社内）
  - **完了条件**（何が揃えば次へ行けるか）
- “やること”は名詞でなく動詞（電話する/メールする/確認する/依頼する）

---

## 「聞けない」前提の安全装置

### 1) 仮置き案の書き方

- 断定しない（会社ルール差がある）
- **一番リスクが低い仮**を提示し、次の確認先をセット

例:

- 仮置き: 「入金期限が不明 → いったん『旅行開始の○日前』を仮とし、社内ルールを確認する」
- 次の確認: 「社内（経理/上長）に“標準の入金締め”を確認」

### 2) エスカレーション（今すぐ相談すべき）トリガー

- 期限が当日/翌日で、在庫や入金が未確定
- キャンセル/変更が発生し、費用が増える可能性がある
- 顧客と仕入先で言っている条件が食い違う

## 品質チェックリスト（作成後）

- [ ] 1画面目で「次の一手（最大3つ）」が見える
- [ ] 次の一手に「期限・相手・完了条件」がある（仮置き可）
- [ ] 進捗が決定→精算のどこかに必ず当てはまっている
- [ ] “未完”の項目が放置されず、質問/一手に接続している
- [ ] 用語がやさしく言い換えられている（初出は解説）
- [ ] 送れる文面テンプレがある（質問が行動に変わる）
- [ ] Lucide のみ（絵文字なし）
- [ ] スマホで読める

## ヒアリングSSOTとの対応

| ヒアリング | 図解での反映 |
|-----------|-------------|
| 新人が知識ゼロ | 用語解説＋工程トラッカー＋完了条件 |
| 一人で作業、聞けない | 質問テンプレ＋仮置き案＋エスカレーショントリガー |
| Excelに工程がある | 「できた/未完」を工程に割り当てて可視化 |
| 伴走してほしい | ミナ×AIの対話で不安→行動へ橋渡し |
