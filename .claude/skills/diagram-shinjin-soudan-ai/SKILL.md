# Diagram Shinjin Soudan AI

**旅行会社の新人（知識ゼロ）**が、一人で作業していても迷子にならないように、
旅行決定〜精算までの流れを「いまの状況 → 次の一手 → 期限 → 確認先」に変換した **単一HTML（Tailwind + Lucide）** を生成する。

## ヒアリングで固めた前提（SSOT）

| 項目 | 内容 |
|------|------|
| 業界・職種 | 旅行会社／営業／指導社員（作成者） |
| 図解を見る人 | 新人（何の知識もない） |
| 利用シーン | 新人が一人で作業していて、誰にも聞けない状況で「壁打ち」したい |
| 既存資産 | 「旅行が決定してから精算までの流れ」は Excel にまとまっている |
| 成功の定義 | 仕事を全く理解していない状態でも、AIが相談相手になり、いまの状況から次の行動を決めて前に進める（伴奏できる） |

## ワークフロー

### Phase 1: ソース読み込み（新人の入力を受け取る）

ユーザー（新人）から、以下のいずれかを受け取る。

- Excelの貼り付け（工程の項目と、できている/できていないの状況）
- 文章での状況メモ（できていること／詰まっていること／期限／顧客情報）

この時点で、**分からない情報は「仮置き」**してよい（ただし Phase 2 で「確認すべき質問」として明示する）。

### Phase 2: 状況を「工程」に割り当てる（決定〜精算）

工程を、最低限この7ブロックに整理する（Excelの項目名が違っても対応づける）。

1. **決定**（旅行内容の確定/社内OK）
2. **見積・条件**（金額、手数料、キャンセル、支払い条件）
3. **手配**（航空/宿/バス/現地、確保、変更対応）
4. **書類**（旅程、最終案内、バウチャー、請求関連）
5. **請求**（請求書発行、入金予定、督促）
6. **実施**（催行、当日連絡、トラブル対応）
7. **精算**（支払/原価確定/粗利確認/完了報告）

### Phase 3: 「次に何をするか」を決める（最優先の一手）

新人が迷子になりやすいので、必ず以下を出す。

- **次の一手（最大3つ）**: 今日やること
- **期限**: いつまでに（なければ仮）
- **相手**: 誰に確認/依頼（顧客、仕入先、社内）
- **完了条件**: 何が揃えば次へ進めるか（証拠/書類/入金など）

### Phase 4: 「確認すべき質問」を生成（聞けない時の壁打ち）

不足情報を、**そのままの質問**にして並べる。質問は3種類に分ける。

- **顧客に聞く**（例: 人数確定、氏名表記、支払い方法）
- **仕入先に聞く**（例: 在庫、変更料、支払期限）
- **社内に聞く**（例: 値付けルール、承認、精算ルール）

さらに「いま聞けない」前提なので、各質問に **代替の仮置き案**（最も安全な仮）を添える。

### Phase 5: 用語翻訳表（知識ゼロ前提）

略語・業界用語・精算用語は、[references/term-dictionary.md](references/term-dictionary.md) の形式で必ず解説する（初出は用語解説ボックス）。

### Phase 6: キャラクター対話（安心して前に進める）

導入・中盤（詰まりポイント）・まとめに、**新人ミナ × 指導社員AI**の短い対話を入れる。

詳細 → [references/character-usage.md](references/character-usage.md)

### Phase 7: HTML生成

1. Tailwind CSS CDN + Lucide（**絵文字禁止**）
2. 配色・コンポーネントは [references/html-structure.md](references/html-structure.md) の「新人伴走用」変数を使用
3. 必須コンポーネント（必ず含める）
   - **次の一手（最大3つ）**
   - **進捗トラッカー（決定→精算）**
   - **いまの状態（できている/未完）一覧**
   - **確認すべき質問（顧客/仕入先/社内）**
   - **用語解説ボックス**

### Phase 8: 批判的レビュー（任意・サブエージェント）

```
Task({
  subagent_type: "generalPurpose",
  description: "図解の批判的レビュー（新人視点）",
  readonly: true,
  prompt: `
    あなたは「旅行会社の新人（知識ゼロ、今ひとりで作業中）」です。
    以下のHTML図解を読んで、これだけで次に進めるかを基準にレビューしてください。
    【HTMLファイルパスまたは本文】{path or excerpt}

    ## レビュー観点
    1. 今日やることが3つ以内で明確か（迷わず着手できるか）
    2. 「いつまでに」「誰に」が抜けていないか（仮置きでも良い）
    3. 用語・略語が置きっぱなしになっていないか
    4. いまの状態（できた/できてない）が自分の入力と一致しているか
    5. 聞けない時の“代替案（仮置き）”があるか

    ## 出力形式
    ### 致命的な問題
    ### 改善すべき点
    ### 良かった点
    ### 総評（5段階＋一言）
  `
})
```

### Phase 9: 納品形態

- 既定: **単一 `index.html`** として保存（ユーザー指定パスがあれば従う）

- [ ] 決定→精算の**進捗トラッカー**がある

### デザイン

- [ ] Lucide のみ（絵文字禁止）
- [ ] スマホでも読める（レスポンシブ）
