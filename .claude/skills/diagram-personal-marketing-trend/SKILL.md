# Diagram Personal Marketing Trend

**SaaS業界のデジタルマーケ担当（自分）**向けに、記事・公式ブログ・社内メモを「運用判断に使える図解ダッシュボード（単一HTML）」へ変換する。

## ヒアリングで固定した前提（SSOT）

| 項目 | 内容 |
|------|------|
| あなたの仕事 | SaaS / デジタルマーケ / 広告運用・データ分析・レポート作成 |
| 読み手 | 自分だけ |
| 課題 | AI系新機能の本質把握が追いつかず、テキストメモは忘れやすい |
| 典型シーン | Google公式のP-MAX仕様変更を読んで判断に2時間かかる |
| ゴール | 図解ストックを見返し、「何か・どう影響するか」を30秒で想起できる状態 |

---

## 出力仕様（固定）

- 出力は **単一HTML**（`index.html`）
- UIは **Tailwind CSS CDN + Lucide**
- 絵文字は禁止
- 1画面目で「重要変更」「運用影響」「次アクション」が分かる構成にする

---

## ワークフロー

### Phase 1: ソース理解

入力（URL、記事本文、メモ）から以下を抽出する。

1. 何が変わったか（仕様差分）
2. なぜ重要か（影響の大きさ）
3. どこに効くか（配信、入札、クリエイティブ、計測、レポート）

### Phase 2: 変更を3層で整理

必ず次の3層で整理する。

- **事実**: 公式発表の変更点
- **解釈**: 実務上の意味
- **行動**: 今日/今週でやること

### Phase 3: 想起用の圧縮

図解の中核を「30秒で思い出せる最小単位」に圧縮する。

- **30秒サマリー**: 1〜2文（変更点 + 運用影響）
- **覚える3点**: 重要論点を3つ固定
- **運用ToDo**: 今日やる1件、今週やる2〜3件

### Phase 4: HTMLに変換

以下のセクションを必須で実装する。

1. 今日の結論（30秒サマリー）
2. 仕様変更マップ（Before / After）
3. 運用影響マトリクス（影響大/中/小 × 領域）
4. 実行アクション（今日 / 今週）
5. 用語メモ（略語や新機能を短く定義）
6. 次回見返しトリガー（「どの場面で再確認するか」）

### Phase 5: 品質確認

- 情報を詰め込みすぎず、5分で読み返せる量にする
- 推測は「仮説」と明記し、事実と混ぜない
- 変更点とToDoが直接つながっていることを確認する

---

## 出力テンプレート（要約フォーマット）

図解作成前に、内部でこの形に整理してからHTML化する。

```markdown
## 変更の要点（30秒）
- [1〜2文で要約]

## 覚える3点
1. [最重要]
2. [次に重要]
3. [運用で効く観点]

## 影響整理
- 影響領域: [配信/入札/クリエイティブ/計測/レポート]
- 影響度: [大/中/小]
- 根拠: [公式情報 or 観測]

## アクション
- 今日: [1件]
- 今週: [2〜3件]
```

---

## 依頼を受けたときの挙動

次のような依頼でこのスキルを優先適用する。

- 「この仕様変更を図解化して」
- 「P-MAXの更新を運用影響つきで整理したい」
- 「AI広告機能の要点をすぐ思い出せる形にしたい」
- 「テキストメモだと忘れるので図で残したい」

---

## 完了条件

- 単一HTMLを生成している
- 30秒サマリーがある
- 覚える3点がある
- 今日/今週の行動が明確
- 見返したときに運用判断へ直結する
